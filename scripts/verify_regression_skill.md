# 스킬 가이드: A 수정 시 B/C 영향 방지를 위한 리그레션 검토 지침

이 파일은 AI 개발 에이전트(Antigravity 등) 및 개발자가 **핵심 API나 백엔드 로직('A')**을 수정할 때, **매장 설정 및 데이터 스키마('B')**와 **E2E 비즈니스 시나리오('C')**에 부작용을 일으키지 않도록 자동 검증하는 절차를 규정한 **스킬 지침**입니다.

이 지침을 읽고 실행(Load)하여 변경 작업의 무결성을 보장하십시오.

---

## 🛠️ 검증 사전 지침 (AI Agent 전용)
작업 대상 소스 코드를 편집하기 전에 다음을 확보하십시오:
1. **의존성 맵 확인**: 수정한 함수나 API 엔드포인트가 `types.ts`, `master_config_v2_그레이스_하이테크_커피.json`, `test_e2e.py`에서 호출되거나 공유되는 구조인지 영향도를 확인합니다.
2. **서버 구동 상태**: 로컬 서버가 구동 중인지(`http://localhost:8000`) 확인합니다. 구동되어 있지 않다면 `run.bat`을 실행해두어야 통합 하네스 검증이 가능합니다.

---

## 🏃 검증 절차 (Verification Steps)

### 1단계: 사전 검증 (Pre-check)
수정을 시작하기 전, 현재 시스템 상태가 정상인지 리그레션 테스트 하네스를 선제 실행합니다.
```powershell
python scripts/verify_regression.py
```
*이 단계에서 이미 실패가 뜬다면, 변경 전 상황에 이미 에러가 존재하므로 먼저 환경 정리가 필요합니다.*

### 2단계: 코드 수정 ('A' 변경 적용)
원하는 대상 코드('A')를 최소 범위로 신중하게 수정합니다. 
수정 시 기존 타입 명세나 함수 시그니처를 함부로 파괴하지 않도록 주의합니다.

### 3단계: 하네스를 통한 사후 검증 (Post-check & Harness execution)
수정이 완료되면 즉시 리그레션 하네스를 가동합니다.
```powershell
python scripts/verify_regression.py
```
* **성공**: 콘솔에 `[SUCCESS] 리그레션 없음!`이 표시되면 검증 완료입니다.
* **실패**: `[FAIL] E2E 테스트 시나리오 실패 검출`이 나타날 경우, 방금 변경한 코드가 B 또는 C에 영향(사이드 이펙트)을 미쳐 정상 상태 전이를 방해하고 있다는 의미이므로 즉시 원인을 추적하거나 변경 사항을 롤백하십시오.

## 🚨 실패 시 대응 지침 (Troubleshooting)
* **API 연결 실패**: `run.bat` 콘솔 창이 닫혔거나 에러가 났는지 확인하고 재부팅하십시오.
* **E2E 테스트 시나리오 실패**: `test_e2e.py`가 출력한 에러 로그에서 어떤 세션 혹은 주문 API가 예외(Exception)나 불일치 데이터를 리턴했는지 추적하십시오.
